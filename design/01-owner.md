# Владелец клиники — Design Prompts (Stitch)

> Роль: Владелец / директор клиники. Полный доступ. Финансы, персонал, филиалы, аналитика, маркетинг, настройки.

---

## 1.1 Дашборд

```
Разработайте современный, профессионально выглядящий интерфейс главного дашборда владельца клиники.

**Визуальный стиль**
Чистый медицинский дизайн. Основной цвет #1E40AF (индиго), фон #F8FAFC, акценты — #10B981 (позитив), #EF4444 (негатив), #F59E0B (предупреждение). Шрифт Inter. Иконки Lucide/Phosphor. Тени мягкие (shadow-sm), скругления 8–12 px. Карточки на белом фоне с border 1 px #E2E8F0.

**Макет и структура**
Sidebar 240 px (сворачивается до 64 px с иконками), sticky header 56 px с поиском, уведомлениями (bell badge), аватаром пользователя и переключателем филиала. Верхняя строка — 5 KPI-карточек в ряд: выручка сегодня / за неделю / за месяц с трендовой стрелкой и процентом, пациентов сегодня, загрузка клиники %, средний чек, новые пациенты. Под KPI — grid виджетов 2–3 колонки: area-chart выручки за 30 дней, donut по отделениям, bar-chart топ-5 услуг, bar-chart топ-5 врачей с аватарами, таблица последних 10 кассовых операций, heatmap записей по часам.

**Элементы UI**
KPI-карточка: иконка в цветном круге, значение крупным шрифтом (24 px semibold), подпись серым, тренд (зелёная/красная стрелка + %). Виджеты с заголовком, кнопкой «...» (dropdown: развернуть, экспорт, настроить). Графики — Recharts/Nivo, палитра индиго-голубая. Таблица кассовых операций — компактная, 10 строк, статус-бейджи (оплачено/возврат/ожидание). Heatmap — сетка часы×дни, градиент от #EFF6FF до #1E40AF.

**UX-аспекты**
Данные загружаются с skeleton-анимацией. KPI-карточки кликабельны — переход к детальному отчёту. Виджеты drag-and-drop для персонализации. Период переключается в header (сегодня/неделя/месяц). Все числа форматируются с разделителем тысяч (пробел). Валюта — UZS.

**Брендинг**
Логотип клиники в верхней части sidebar. Название клиники рядом с лого (скрывается при свёртке). Цвета соответствуют корпоративной палитре #1E40AF.

**Доступность**
Контраст текста WCAG AA. Все интерактивные элементы доступны с клавиатуры. Aria-labels для графиков. Tooltips при hover на данных.

**Тренды**
Bento-grid layout. Glassmorphism для модальных окон. Микроанимации при загрузке данных (fade-in, counter-up для цифр). Subtle gradient в header sidebar.
```

## 1.2 Финансы — Доходы

```
Разработайте современный, профессионально выглядящий интерфейс раздела доходов клиники.

**Визуальный стиль**
Основной #1E40AF, фон #F8FAFC, позитив #10B981, негатив #EF4444. Шрифт Inter. Иконки Lucide/Phosphor. Карточки белые с shadow-sm и border #E2E8F0, скругления 8–12 px.

**Макет и структура**
Sidebar 240→64 px, sticky header 56 px. Вверху — фильтры в одну строку: период (date range picker), филиал (select), отделение (select), кнопка «Применить», кнопка «Экспорт» (Excel/PDF). Под фильтрами — 5 KPI-карточек: общий доход, наличные, карта, перевод, страховые. Ниже — area-chart выручки по дням с переключателем группировки (день/неделя/месяц). Под графиком — DataTable доходов.

**Элементы UI**
KPI-карточки с иконкой метода оплаты, суммой крупно, процентом от общего мелко. Date range picker с пресетами (сегодня/неделя/месяц/квартал/год/произвольный). Area-chart — заливка градиентом #1E40AF→transparent, точки при hover. DataTable колонки: дата, номер счёта, пациент (ФИО), услуга, врач, сумма, способ оплаты (цветной badge), статус (badge: оплачен/частично/возврат). Пагинация, сортировка по всем колонкам, поиск.

**UX-аспекты**
Фильтры применяются с debounce. Skeleton при загрузке. Клик на строку таблицы — slide-over с деталями счёта. Сумму можно копировать по клику. Экспорт формирует файл в фоне с уведомлением. Числа с разделителем тысяч, валюта UZS.

**Брендинг**
Палитра корпоративная. Индиго-тона для графиков. Лого в sidebar.

**Доступность**
WCAG AA контраст. Tab-навигация по таблице. Screen reader friendly заголовки колонок. Aria-sort на колонках.

**Тренды**
Sticky table header при скролле. Inline-фильтры с chips. Smooth chart transitions при смене периода.
```

## 1.3 Финансы — Расходы

```
Разработайте современный, профессионально выглядящий интерфейс раздела расходов клиники.

**Визуальный стиль**

**Макет и структура**
Sidebar 240→64 px, header 56 px. Вверху — filter pills категорий: зарплаты, аренда, расходники, оборудование, маркетинг, налоги (цветные, toggle). Справа — date range picker и кнопка «+ Добавить расход». Основная область: слева — таблица расходов (70%), справа — pie-chart по категориям (30%). Под ними — bar-chart ежемесячного тренда расходов.

**Элементы UI**
Filter pills — rounded-full, каждая категория со своим цветом и иконкой, активная — заливка, неактивная — outline. Inline-добавление расхода без модального окна. Drag файла документа прямо в строку. Удаление через swipe или иконку с confirm. Pills фильтруют мгновенно. Суммы форматированы, UZS.

**Брендинг**
Каждая категория расходов имеет уникальный цвет из расширенной палитры, гармонирующий с основным #1E40AF.

**Доступность**
Контраст WCAG AA. Filter pills доступны с клавиатуры (toggle role). Aria-labels для графиков.

**Тренды**
Inline editing. Animated pie transitions. Subtle hover effects на строках таблицы.
```

## 1.4 Финансы — P&L

```
Разработайте современный, профессионально выглядящий интерфейс отчёта о прибылях и убытках (P&L) клиники.

**Визуальный стиль**
Основной #1E40AF, фон #F8FAFC, доходы #10B981, расходы #EF4444, маржа — градиент. Шрифт Inter. Карточки белые, shadow-sm.

**Макет и структура**
Sidebar 240→64 px, header 56 px. Вверху — переключатель периода (месяц/квартал/год), toggle «Сравнение с предыдущим периодом», кнопка «Экспорт». Основная область: dual area-chart — доходы (зелёная заливка) vs расходы (красная заливка) с зоной прибыли между ними. Под графиком — P&L таблица по месяцам.

**Элементы UI**
Dual area-chart: две области с прозрачной заливкой, зона прибыли подсвечена зеленоватым. Tooltip при hover — доход, расход, прибыль, маржа%. P&L таблица: строки — месяцы, колонки — доходы (по категориям), итого доходов, расходы (по категориям), итого расходов, прибыль, маржа%. Итоговая строка bold. Маржа% — progress bar внутри ячейки. При включении сравнения — дополнительная колонка с дельтой (стрелка + %).

**UX-аспекты**
Toggle сравнения анимированно добавляет колонки. Hover на строке — подсветка. Клик на месяц — drill-down к детальной разбивке. Экспорт в Excel/PDF. Числа с разделителями, UZS.

**Брендинг**
Зелёно-красная бинарность на корпоративном индиго-фоне графиков. Профессиональный финансовый стиль.

**Доступность**
WCAG AA. Таблица с scope headers. Цвета дублируются текстовыми метками (+/−). Aria-labels.

**Тренды**
Gradient fills в графиках. Animated number counters. Smooth column transitions при toggle сравнения.
```

## 1.5 ДМС / Страховые

```
Разработайте современный, профессионально выглядящий интерфейс управления страховыми компаниями и ДМС.

**Визуальный стиль**
Основной #1E40AF, фон #F8FAFC, статусы: активный #10B981, истекает #F59E0B, неактивный #EF4444. Шрифт Inter. Иконки Lucide/Phosphor.

**Макет и структура**
Sidebar 240→64 px, header 56 px. Верхняя секция — карточки страховых компаний (grid 3–4 колонки). Под ними — tabs: «Реестры», «Акты сверки», «Проверка покрытия». Каждый таб — своя таблица/форма.

**Элементы UI**
Карточка СК: логотип компании, название, номер договора, срок действия (progress bar до окончания), баланс/лимит, статус-badge. Hover — подсветка, клик — детали. Таблица реестров: номер, период, кол-во услуг, сумма, статус (badge: черновик/отправлен/принят/отклонён/оплачен). Акты сверки — таблица с кнопкой генерации. Проверка покрытия — форма: выбор СК, номер полиса, кнопка «Проверить», результат — карточка с лимитами и доступными услугами.

**UX-аспекты**
Карточки СК сортируются по статусу (истекающие первыми). Уведомление за 30 дней до окончания договора. Реестр формируется автоматически из оказанных услуг. Проверка покрытия — real-time запрос.

**Брендинг**
Корпоративная палитра. Логотипы СК отображаются в их фирменных цветах на нейтральном фоне.

**Доступность**
WCAG AA. Cards доступны как focusable элементы. Tabs с aria-selected. Форма с labels.

**Тренды**
Card-based layout. Status indicators с pulse-анимацией для истекающих. Slide-over для деталей.
```

## 1.11 Аналитика — Врачи

```
Разработайте современный, профессионально выглядящий интерфейс аналитики по врачам клиники.

**Визуальный стиль**
Основной #1E40AF, фон #F8FAFC, рейтинг #F59E0B (звёзды), позитив #10B981. Шрифт Inter. Аватары круглые. Карточки белые shadow-sm.

**Макет и структура**
Sidebar 240→64 px, header 56 px. Переключатель вида: карточки / таблица. Фильтры: специальность (select), отделение, период. Сортировка: по выручке / по приёмам / по рейтингу / по загрузке. Основная область — grid карточек врачей или DataTable.

**Элементы UI**
Карточка врача: аватар 64 px, ФИО, специальность badge, 4 метрики в ряд (приёмов, выручка, средний чек, загрузка% progress bar), рейтинг 5 звёзд с числом. Таблица: те же данные в колонках, sortable. Клик на врача — slide-over с детализацией: line-chart помесячных трендов (приёмы + выручка dual axis), разбивка по услугам pie-chart, последние отзывы.

**UX-аспекты**
Сортировка анимированная (layout shift). Карточки/таблица сохраняют выбор в localStorage. Загрузка% — цветовой индикатор (зелёный < 70, жёлтый 70–90, красный > 90). Поиск по ФИО.

**Брендинг**
Аватары врачей — круглые с тонкой индиго-рамкой. Корпоративные badge-цвета специальностей.

**Доступность**
WCAG AA. Звёзды рейтинга — aria-label с числовым значением. Toggle вида доступен с клавиатуры.

**Тренды**
Card/table view toggle. Animated sort. Progress bars в карточках. Slide-over detail panel.
```

WCAG AA. Графики с aria-labels. Таблица с headers scope. Filter pills — role="checkbox".

**Тренды**
Inline sparklines. Animated bars. Category pills с count badges. Slide-over details.

Автосохранение с debounce и toast «Сохранено». Dangerous actions (сброс настроек) — confirm dialog с вводом текста. 2FA — пошаговый flow с QR и верификацией. Color picker — live preview. Валидация форм inline.

**Брендинг**
Настройки брендинга позволяют кастомизировать внешний вид. Preview обновляется в реальном времени.

**Доступность**
WCAG AA. Vertical tabs с aria-orientation="vertical". Все inputs с labels. Color picker с text input fallback.

**Тренды**
Vertical tab navigation. Auto-save. Live preview для брендинга. Confirmation patterns для опасных действий.
