# Главный врач — Design Prompts (Stitch)

> Роль: Главный врач. Контролирует качество медицинской помощи, управляет врачами, протоколами лечения, расписанием, отделениями, назначениями и отчётностью.

> Дизайн-система: цвета #1E40AF (primary blue), #F8FAFC (фон), #10B981 (зелёный/успех), #EF4444 (красный/ошибка), #F59E0B (жёлтый/предупреждение). Шрифт Inter: заголовки 24px bold, подзаголовки 16px semibold, тело 14px regular. Иконки Lucide/Phosphor. Sidebar 240px→64px collapsible + sticky header 56px (поиск, уведомления, язык RU/UZ, аватар). Desktop-first, адаптивный web.

## 2.1 Дашборд главного врача

```
Разработайте современный, профессионально выглядящий интерфейс дашборда главного врача клиники в медицинской информационной системе.

Визуальный стиль:
- Чистый, медицинский дизайн с акцентом на оперативный мониторинг и контроль работы клиники в реальном времени
- Цветовая палитра: основной #1E40AF для ключевых акцентов и навигации, фон #F8FAFC для контентной области, #10B981 для позитивных показателей и нормальных статусов, #F59E0B для предупреждений и отклонений от нормы, #EF4444 для критических значений и жалоб
- Обильное белое пространство между виджетами — каждый виджет на карточке с тонкой тенью shadow-sm и скруглением border-radius 12px
- Типографика Inter: заголовки виджетов 16px semibold в #1E293B, значения KPI 28px bold, подписи 12px regular в #64748B
- Контентная область с мягким фоном #F8FAFC, карточки на белом #FFFFFF, визуальное отделение через тени а не через бордеры
- Общая атмосфера — командный центр руководителя, не перегруженный но информативный

Макет и структура:
- Левый sidebar (240px, collapsible до 64px): иконки + текст пунктов — Дашборд (активный, подсветка #1E40AF фон с opacity 10%), Расписание, Расписание врачей, Мониторинг, Протоколы, Контроль качества, Медкарты, Статистика врачей, Отделения, Назначения, Отчёты. Внизу — аватар пользователя с ФИО и ролью «Главный врач»
- Sticky header 56px: слева — breadcrumb «Дашборд», по центру — global search input с placeholder «Поиск врача, пациента, кабинета...» (иконка Search, border-radius 8px, фон #F1F5F9), справа — иконка колокольчика с red badge (количество), toggle языка RU/UZ (segmented mini control), аватар 32px с dropdown
- KPI-ряд верхний — 5 карточек в одну строку с равной шириной: (1) Активные приёмы сейчас — крупное число, иконка Activity, синий акцент; (2) Завершено сегодня — число, иконка CheckCircle, зелёный акцент; (3) Врачей онлайн — число/общее (напр. 18/24), иконка Users, синий акцент; (4) Среднее время ожидания — минуты, иконка Clock, цвет по условию (<10мин зелёный, 10-20 жёлтый, >20 красный); (5) Жалобы — число с badge «новые», иконка AlertTriangle, красный акцент если >0
- Каждая KPI-карточка содержит: иконку в цветном круге 40px слева, значение крупным шрифтом, подпись мелким, внизу — trend indicator (стрелка вверх/вниз + процент изменения за период, зелёный для позитивного тренда, красный для негативного)
- Основная область — grid 2 колонки ниже KPI:
  - Левая колонка (60%): (1) Виджет «Таймлайн приёмов в реальном времени» — горизонтальная шкала времени с 8:00 до 20:00, на ней цветные блоки текущих приёмов по врачам, маркер «сейчас» (вертикальная пунктирная красная линия), hover показывает tooltip (врач, пациент, время, кабинет). (2) Виджет «Загрузка отделений» — горизонтальные progress bars по каждому отделению (название слева, bar справа, процент на конце), цвет bar по уровню (<50% зелёный, 50-80% синий, >80% жёлтый, >95% красный), сортировка по убыванию загрузки
  - Правая колонка (40%): (1) Виджет «Статусы врачей» — grid карточек 2×N, каждая мини-карточка: аватар 36px, ФИО, специальность мелким, status dot (зелёный=свободен, синий=на приёме, серый=перерыв, красный=задержка), если на приёме — имя текущего пациента мелким шрифтом. (2) Виджет «Последние медзаписи» — compact list: время, врач, пациент, диагноз (truncated), статус badge (Черновик жёлтый, Подписана зелёный), максимум 8 записей с кнопкой «Все записи». (3) Виджет «Оценка качества» — gauge/спидометр от 0 до 100 с текущим значением крупно в центре, цветовые зоны (красный 0-40, жёлтый 40-70, зелёный 70-100), подпись «Quality Score» и trend vs прошлый месяц

Элементы UI:
- KPI-карточки с цветными иконками в мягких кругах (цвет иконки с opacity 15% для круга-фона), hover — subtle elevation increase
- Status dots (8px круг) с цветовой кодировкой: зелёный #10B981 свободен, синий #3B82F6 на приёме, серый #94A3B8 перерыв, красный #EF4444 задержка
- Trend arrows — стрелки ChevronUp/ChevronDown с процентом, зелёный для положительного тренда, красный для отрицательного
- Progress bars для загрузки отделений — height 8px, rounded-full, с animated fill при загрузке
- Timeline горизонтальный — цветные прямоугольники с rounded corners, полупрозрачные для будущих приёмов, непрозрачные для текущих и прошедших
- Gauge/спидометр — SVG дуга с gradient (красный→жёлтый→зелёный), стрелка-указатель, число в центре 32px bold
- Compact table rows с alternating bg (#FFFFFF и #F8FAFC), hover row highlight #F1F5F9
- Badge для жалоб — pill с числом, красный фон #FEE2E2, красный текст #DC2626, pulse animation если >0

UX-аспекты:
- Клик на KPI-карточку «Активные приёмы» → переход на экран 2.4 Мониторинг
- Клик на карточку врача в grid → переход на расписание этого врача (2.3)
- Клик на строку в «Последних медзаписях» → переход на просмотр записи (2.7)
- Клик на gauge качества → переход на Контроль качества (2.6)
- Timeline приёмов автоматически обновляется каждые 30 секунд с плавной анимацией transition 0.3s
- Виджеты загружаются с skeleton placeholders (серые мерцающие прямоугольники) до получения данных
- Drag-and-drop для перестановки виджетов (настройка дашборда), сохранение layout в профиле пользователя
- При наведении на отделение в bars → tooltip с детализацией: врачей на смене, записей всего, свободных слотов

Брендинг:
- Логотип клиники в верхней части sidebar (при collapsed — только иконка)
- Название системы «МИС» рядом с логотипом, исчезает при collapsed sidebar
- Цвета sidebar: фон #1E293B (тёмный), текст #CBD5E1, активный пункт — фон #1E40AF/10, текст #FFFFFF, иконка #60A5FA
- Профессиональный медицинский стиль без избыточной декоративности

Доступность:
- Все KPI-карточки имеют aria-label с полным описанием значения и тренда, например «Активные приёмы: 12, рост 5% за день»
- Status dots продублированы текстовыми labels (sr-only) для скринридеров
- Gauge имеет aria-valuenow, aria-valuemin, aria-valuemax и aria-label
- Timeline блоки кликабельны с keyboard (Tab + Enter), focus ring visible
- Progress bars с aria-valuenow и aria-label с названием отделения и процентом
- Контраст текста минимум 4.5:1 по WCAG AA, все интерактивные элементы с focus-visible outline 2px #1E40AF
- Color-coding всегда дублируется текстом или иконкой — никогда только цветом

Тренды:
- Glassmorphism на KPI-карточках: background rgba(255,255,255,0.7), backdrop-filter blur(12px), тонкий border rgba(255,255,255,0.3)
- Subtle pulse animation на badge жалоб — мягкое мерцание красного кольца каждые 2 секунды
- Smooth data transitions — числа в KPI анимируются при обновлении (count-up эффект)
- Micro-interactions: hover на карточку врача — scale(1.02) с transition 150ms ease
- Gradient accent на gauge — плавный переход красный→жёлтый→зелёный в SVG дуге
- Skeleton loading вместо спиннеров — shimmer effect на placeholder блоках
- Timeline блоки с subtle shadow при hover и smooth tooltip fade-in 200ms
```

## 2.2 Управление расписанием

```
Разработайте современный, профессионально выглядящий интерфейс управления расписанием всех врачей клиники для главного врача в медицинской информационной системе.

Визуальный стиль:
- Calendar-oriented дизайн с плотной но читаемой сеткой, акцент на цветовую кодировку по отделениям
- Цветовая палитра: основной #1E40AF, фон #F8FAFC, каждое отделение имеет свой цвет — Терапия #3B82F6 синий, Хирургия #EF4444 красный, Педиатрия #10B981 зелёный, Стоматология #F59E0B жёлтый, Гинекология #A855F7 фиолетовый, Офтальмология #06B6D4 бирюзовый, остальные — палитра пастельных тонов
- Тонкие borders #E2E8F0 для сетки, скруглённые ячейки при hover, выделенные ячейки с заливкой цветом отделения с opacity 15%
- Типографика Inter: заголовки дней 14px semibold, имена врачей 13px medium, время в ячейках 12px regular, compact layout для максимальной информативности
- Фон сетки #FFFFFF, header строки (дни недели) — #F1F5F9, group headers отделений — цвет отделения с opacity 8% и left-border 3px цветом отделения
- Общее ощущение — диспетчерский пульт планирования ресурсов

Макет и структура:
- Sticky header 56px: breadcrumb «Расписание > Управление», справа — стандартные элементы (поиск, уведомления, язык, аватар)
- Toolbar под header (height 48px, bg white, border-bottom): слева — segmented control День|Неделя|Месяц (Неделя активна по умолчанию, pill-style с animated slider), навигация неделями (ChevronLeft, «10–16 марта 2026», ChevronRight), кнопка «Сегодня» ghost. По центру — filter chips: dropdown «Отделение» (мульти-выбор с чекбоксами и цветными dots отделений), dropdown «Врач» (autocomplete search), toggle «Только свободные слоты». Справа — кнопка «Шаблоны расписаний» (outline, иконка Layout), кнопка «Массовое назначение» (primary, иконка CalendarPlus)
- Основная область — weekly calendar grid: горизонтально 7 дней (Пн-Вс), вертикально — врачи сгруппированные по отделениям
  - Group header отделения: sticky при вертикальном скролле, цветной left-border, название отделения bold, badge с количеством врачей, кнопка expand/collapse (ChevronDown)
  - Строка врача: слева фиксированная колонка 200px (аватар 28px + ФИО 13px medium + специальность 11px muted), справа — 7 ячеек по дням
  - Ячейка дня: если есть расписание — цветная заливка (цвет отделения, opacity 12%), текст «09:00–17:00», мелким «каб. 305», число записей/свободных «12/4»; если выходной — серый фон #F1F5F9; если отпуск — синий фон #DBEAFE с иконкой Palmtree; если больничный — оранжевый #FEF3C7 с иконкой Thermometer
  - При hover на ячейку — elevation, при клике — popup с деталями (время работы, кабинет, перерывы, записи, кнопки Редактировать/Удалить)
- Drag-and-drop: можно перетащить ячейку расписания врача на другой день или другого врача для переназначения; при перетаскивании — ghost element с opacity 50%, зона drop подсвечивается синим dashed border; при drop — confirmation modal «Перенести расписание Dr. Иванова с Пн на Ср?»
- Модальное окно «Шаблоны расписаний»: список шаблонов (карточки — название, описание, preview мини-календарь 7 дней с цветными/серыми ячейками), кнопка «+ Создать шаблон». Создание: название, выбор рабочих дней (чекбоксы Пн-Вс), время начала/окончания для каждого рабочего дня, перерыв (время + длительность), кабинет default. Preview шаблона. Применение: выбор врача(ей) multi-select + период (date range) + кнопка «Применить» с preview результата
- Модальное окно «Массовое назначение»: step 1 — выбрать врачей (чекбоксы из списка, фильтр по отделению), step 2 — выбрать шаблон или задать параметры вручную, step 3 — выбрать период, step 4 — preview + confirm

Элементы UI:
- Segmented control с pill slider (animated transition 200ms, bg #1E40AF для активного, text white)
- Filter chips — pill-shape с цветной dot слева (цвет отделения), кнопка X для сброса, dropdown с чекбоксами
- Grid cells с rounded-lg при hover, transition shadow + transform 150ms
- Group headers — sticky (z-index: 10) с backdrop-filter blur при скролле
- Drag ghost — semi-transparent карточка с dashed border, drop zone — blue dashed outline
- Mini calendar в шаблонах — 7 маленьких квадратов (цветные рабочие, серые выходные)
- Confirmation modals — с чётким описанием действия, кнопки Cancel (ghost) и Confirm (primary)
- Badge pills для подсчёта — rounded-full, font-size 11px, bg цвета отделения с opacity 15%

UX-аспекты:
- Drag-and-drop для быстрого переназначения расписания между врачами и днями с обязательным confirmation
- Conflict detection: если при drag-drop целевой кабинет занят — warning toast с предложением выбрать другой кабинет
- Bulk selection: Shift+Click на ячейки для выделения нескольких, затем действия через context toolbar (Удалить, Скопировать, Применить шаблон)
- Copy week: кнопка «Скопировать неделю» → выбор целевой недели → preview → apply
- Undo/Redo для последних действий (Ctrl+Z/Ctrl+Y) с toast notification «Отменено: перенос расписания Dr. Иванова»
- Фильтры применяются мгновенно с fade transition, скрывая несоответствующие строки
- Клик на имя врача в левой колонке → переход на экран 2.3 (расписание конкретного врача)
- Keyboard navigation: стрелки для перемещения по grid, Enter для открытия ячейки, Escape для закрытия popup

Брендинг:
- Sidebar активный пункт «Расписание» с подсветкой
- Цвета отделений consistent по всей системе — одинаковые на всех экранах
- Профессиональный календарный стиль, напоминающий Google Calendar / resource planning tools

Доступность:
- Grid navigable через keyboard: Tab перемещает между ячейками, Enter открывает детали, Arrow keys для навигации внутри grid
- Цветовая кодировка отделений дублируется текстовым названием отделения в group header и в tooltip
- Drag-and-drop имеет keyboard альтернативу: выделить ячейку → Ctrl+X → навигация к цели → Ctrl+V
- Все modals trap focus внутри, закрываются по Escape
- Screen reader озвучивает: «Строка: Иванов И.И., Терапевт. Понедельник: рабочий день 09:00–17:00, кабинет 305, 12 записей из 16»
- Focus ring 2px #1E40AF на всех интерактивных элементах

Тренды:
- Soft color fills для ячеек — пастельные тона отделений с opacity 10-15%, не утомляющие глаза при длительной работе
- Smooth animated transitions при переключении недель (slide left/right 300ms ease)
- Subtle elevation на hover ячейки — shadow переход от shadow-none к shadow-md за 150ms
- Glassmorphism на toolbar: bg rgba(255,255,255,0.8), backdrop-filter blur(8px), при скролле контент виден под toolbar
- Drag ghost с subtle scale(1.05) и rotation(-2deg) для natural feel
- Skeleton loading для grid при загрузке данных новой недели
```

## 2.3 Расписание врача

```
Разработайте современный, профессионально выглядящий интерфейс просмотра и настройки расписания конкретного врача для главного врача в медицинской информационной системе.

Визуальный стиль:
- Детальный calendar-view дизайн с акцентом на слоты приёма, перерывы и конфигурацию рабочего дня
- Цветовая палитра: основной #1E40AF, фон #F8FAFC, слоты записанные #DBEAFE (синий light), слоты свободные #FFFFFF с border dashed #CBD5E1, перерывы #F1F5F9 с pattern (диагональные линии), заблокированные #FEE2E2 (красный light)
- Чёткая grid-структура с временной шкалой слева, дни недели сверху, тонкие линии #E2E8F0 разделяют временные интервалы
- Типографика Inter: имя врача 24px bold в header, дни недели 14px semibold, время 12px regular monospace, имена пациентов 13px medium
- Карточки слотов с rounded-lg, мягкие тени shadow-sm, цветовая дифференциация по статусу

Макет и структура:
- Header секция (bg white, border-bottom, padding 16px 24px): слева — аватар врача 48px + ФИО 20px bold + специальность badge (pill, цвет отделения) + отделение текст muted; справа — кнопки: «Копировать неделю» (outline, иконка Copy), «Печать» (outline, иконка Printer), «Настройки слотов» (outline, иконка Settings), «Сохранить изменения» (primary, иконка Save, disabled пока нет изменений)
- Навигация недели: ChevronLeft, «10–16 марта 2026» bold, ChevronRight, кнопка «Текущая неделя» ghost, mini date picker (иконка Calendar)
- Основная область — weekly grid (fullwidth):
  - Левая колонка (width 60px, sticky): временная шкала с 08:00 до 20:00, шаг 30 минут, каждый час — bold, полчаса — muted, горизонтальные линии через всю ширину grid
  - 6 или 7 колонок дней (Пн–Сб или Пн–Вс): header каждого дня — день недели + дата + суммарная информация (записей/слотов, напр. «8/12»)
  - Слоты отображаются как блоки в соответствующих временных интервалах:
    - Записанный слот: синий bg #DBEAFE, left-border 3px #3B82F6, текст «09:00 — Иванова М.А.» + тип приёма мелким
    - Свободный слот: белый bg, dashed border #CBD5E1, текст «09:30 — свободно» muted italic
    - Перерыв: серый bg #F1F5F9 с diagonal stripe pattern, текст «Перерыв 13:00–14:00» centered
    - Заблокированный: красный bg #FEE2E2, текст «Заблокировано» с иконкой Lock
  - Кабинет отображается pill badge в правом верхнем углу каждого дня «каб. 305»
- Правая панель «Настройки слотов» (slide-over 360px, появляется при клике на кнопку Settings):
  - Длительность слота: select 10/15/20/30/45/60 мин (текущее значение выделено)
  - Время начала / окончания рабочего дня: time pickers
  - Перерывы: список (время начала + длительность) с кнопкой «+ Добавить перерыв»
  - Кабинет по умолчанию: select из списка свободных кабинетов
  - Максимум пациентов в день: number input
  - Preview: мини-визуализация дня с применёнными настройками
  - Кнопки: «Применить на эту неделю», «Применить на все будущие недели»
- Модальное окно «Копировать неделю»: source неделя (текущая, readonly), target — date range picker для целевого периода, чекбоксы — копировать записи/только структуру, preview результата, кнопки Cancel/Copy

Элементы UI:
- Slot blocks — прямоугольники с rounded-md, high 30-60px в зависимости от длительности, left-border 3px цветной для статуса
```
