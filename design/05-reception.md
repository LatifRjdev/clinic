# Рецепшен (Администратор) — Design Prompts (Stitch)

> Роль: Сотрудник рецепшена. Записывает пациентов, регистрирует визиты, принимает оплату, управляет очередью, печатает документы, отправляет уведомления.

---

## 5.1 Дашборд рецепшена

```
Разработайте современный, профессионально выглядящий интерфейс операционного дашборда для сотрудника рецепшена клиники в медицинской информационной системе.

Визуальный стиль:
- Операционный, быстро считываемый дизайн с акцентом на крупные action-кнопки и ключевые метрики
- Основная палитра: primary #1E40AF (синий), фон #F8FAFC (светло-серый), успех #10B981 (зелёный), ошибка #EF4444 (красный), предупреждение #F59E0B (оранжевый)
- Статусы записей: записан #3B82F6, ожидает #F59E0B, на приёме #10B981, завершён #94A3B8, не явился #EF4444
- Плотный, но визуально чистый layout — максимум информации без перегрузки; белые карточки на светло-сером фоне с мягкими тенями (shadow-sm)
- Типографика Inter: заголовки 24px bold, подзаголовки 16px semibold, основной текст 14px regular
- Иконки Lucide/Phosphor — линейный стиль, 20-24px, stroke-width 1.5

Макет и структура:
- Сворачиваемый sidebar слева (240px → 64px): пункты меню — Дашборд, Запись, Пациенты, Расписание, Касса, Очередь, Документы, Уведомления; иконка + текст, active state — синий фон #EFF6FF с текстом #1E40AF
- Sticky header 56px: слева логотип клиники и название, по центру глобальный поиск (иконка Search, placeholder "Поиск пациента, врача..."), справа — иконка колокольчика с badge-счётчиком уведомлений, переключатель языка RU/UZ (dropdown), аватар пользователя с dropdown-меню
- Блок быстрых действий (grid 2×2 крупных кнопок, высота 80px каждая): "Записать пациента" — primary #1E40AF, иконка CalendarPlus, крупный текст 18px bold; "Регистрация" — secondary outline #1E40AF, иконка UserPlus; "Касса" — зелёный #10B981, иконка Banknote; "Очередь" — оранжевый #F59E0B, иконка Users. Hover-эффект — subtle elevation (shadow-md), active — darken 10%
- KPI-ряд (4 карточки в строку, равная ширина): "Записей сегодня" — число крупно 32px bold + иконка CalendarCheck синяя; "Ожидают сейчас" — число + иконка Clock оранжевая; "Принято" — число + иконка CheckCircle зелёная; "Не явились" — число + иконка XCircle красная. Каждая карточка — белый фон, border-radius 12px, padding 16px, иконка слева в цветном круглом бейдже 40×40px
- Компактный timeline записей на сегодня: вертикальная лента слева (линия #E2E8F0), точки-маркеры по статусу (цвет статуса), справа от точки — строка: время (14px semibold), ФИО пациента (ссылка), врач, услуга, статус badge (pill). Показывать ближайшие 8-10 записей, кнопка "Все записи →" внизу
- Блок алертов справа или над timeline: карточки-предупреждения — "Приём через 15 мин" (синий left-border, иконка Clock), "Опаздывает >10 мин" (красный left-border, иконка AlertTriangle), "Новая онлайн-запись" (зелёный left-border, иконка Globe). Каждый алерт: текст + ФИО + время + кнопка быстрого действия (Позвонить, Подтвердить)

Элементы UI:
- Крупные action-кнопки (минимум 80px высота) с иконкой слева и текстом, border-radius 12px, font-weight 600
- KPI-карточки с цветными иконками в круглых бейджах, число 32px bold, подпись 12px text-gray-500
- Status badges — pill-shaped (border-radius 9999px), padding 4px 12px, font-size 12px semibold, цвет текста + светлый фон статуса
- Timeline с вертикальной линией и цветными точками-маркерами 12px
- Alert-карточки с цветным left-border (4px), иконкой, текстом и inline-кнопкой действия
- Notification badge — красный круг с числом на иконке колокольчика

UX-аспекты:
- Главные действия доступны в один клик с дашборда — кнопки всегда видны без скролла (above the fold)
- KPI обновляются в реальном времени (WebSocket), числа анимируются при изменении (count-up)
- Timeline подсвечивает ближайшую запись — выделение фоном #EFF6FF
- Алерты появляются автоматически за 15 минут до приёма и при опоздании >10 минут
- Горячие клавиши: Ctrl+N — новая запись, Ctrl+R — регистрация, Ctrl+K — поиск пациента
- При клике на запись в timeline открывается popover с деталями и быстрыми действиями

Брендинг:
- Логотип клиники в header слева, 32px высота
- Фирменный синий #1E40AF как основной акцент
- Нейтральный профессиональный медицинский стиль — белый, серый, синий
- Минимум декоративных элементов — фокус на функциональности

Доступность:
- Кнопки быстрых действий минимум 80×80px — удобно для touch и мыши
- Все статусы имеют цвет + текст + иконку (не только цвет)
- KPI-числа с aria-label ("Записей сегодня: 24")
- Алерты с role="alert" и aria-live="polite"
- Контраст текста на цветных бейджах не менее 4.5:1
- Полная клавиатурная навигация по action-кнопкам и timeline

Тренды:
- Glassmorphism на блоке быстрых действий — полупрозрачный фон с backdrop-blur
- Плавная анимация count-up при загрузке KPI-чисел
- Subtle elevation (shadow-md) на hover кнопок быстрых действий
- Микроанимация пульсации на алертах "Опаздывает"
- Smooth scroll timeline с fade-out внизу
```

## 5.2 Запись пациента на приём (Wizard)

```
Разработайте современный, профессионально выглядящий интерфейс пошагового мастера записи пациента на приём для рецепшена в медицинской информационной системе.

Визуальный стиль:
- Wizard-дизайн с горизонтальным progress indicator — чистый, направляющий, с минимумом отвлекающих элементов
- Основная палитра: primary #1E40AF, фон #F8FAFC, акцент подтверждения #10B981, доступные слоты #3B82F6, занятые слоты #E2E8F0
- Каждый шаг занимает центральную область (max-width 720px), контент шага в белой карточке с shadow-sm и border-radius 16px
- Типографика Inter: заголовок шага 24px bold, описание 14px regular text-gray-500, labels 14px semibold
- Иконки Lucide/Phosphor для каждого шага и элементов навигации

Макет и структура:
- Progress indicator вверху (ниже sticky header): горизонтальная полоса с 4 пронумерованными кругами (36px) и соединительными линиями. Шаги: 1 — Пациент (иконка User), 2 — Услуга и врач (иконка Stethoscope), 3 — Дата и время (иконка Calendar), 4 — Подтверждение (иконка CheckCircle). Пройденный шаг — круг заполнен #1E40AF + белая галочка, текущий — круг #1E40AF обводка + число, будущий — серый #D1D5DB + число. Линия между пройденными — #1E40AF, между будущими — #E2E8F0
- Шаг 1 — Выбор пациента: крупная строка поиска (48px высота, иконка Search, placeholder "Введите ФИО, телефон или номер карты..."). Autocomplete dropdown появляется после 2 символов: каждый результат — строка с круглым аватаром 40px (фото или инициалы на цветном фоне), ФИО bold, телефон, дата рождения, дата последнего визита gray. При выборе — карточка пациента отображается ниже (аватар, ФИО, телефон, возраст). Если пациент не найден — кнопка "+ Зарегистрировать нового пациента" (secondary outline) внизу dropdown
- Шаг 2 — Услуга и врач: слева — select "Категория услуг" (dropdown с иконками категорий), ниже список услуг выбранной категории (radio-list с названием, длительностью, ценой). Справа — карточки доступных врачей (после выбора услуги): фото 56px круглое, ФИО bold, специализация gray, ближайший свободный слот — badge зелёный "Сегодня 14:30" или серый "Завтра 10:00". Hover на карточку — border #1E40AF, клик — выбрано (filled border + галочка)
- Шаг 3 — Дата и время: двухпанельный layout. Слева — calendar picker (месячный вид): дни с доступными слотами отмечены зелёной точкой под числом, прошедшие дни — серые disabled, выбранный день — синий круг #1E40AF. Справа — сетка доступных слотов для выбранного дня: кнопки-pills (padding 8px 16px, border-radius 8px) с временем. Свободные — #EFF6FF фон + #1E40AF текст, занятые — #F1F5F9 фон + #94A3B8 текст (disabled), выбранный — #1E40AF фон + белый текст. Над сеткой: инфо "Врач: ..., Кабинет: ..., Длительность: 30 мин"
- Шаг 4 — Подтверждение: summary-карточка с двумя колонками в рамке #E2E8F0 с border-radius 12px. Строки: Пациент (аватар + ФИО), Врач (фото + ФИО + специализация), Услуга (название + цена), Дата и время (форматировано), Кабинет. Каждая строка — label gray слева, значение bold справа. Ниже — checkbox "Отправить SMS-напоминание" (checked по умолчанию), checkbox "Отправить WhatsApp-напоминание". Кнопки: "Записать" (primary #1E40AF, 48px высота, full-width), "Назад" (ghost, слева)
- Навигация: кнопки "Назад" (ghost) и "Далее" (primary) внизу каждого шага. "Далее" disabled пока не выбран элемент на текущем шаге

Элементы UI:
- Progress bar с пронумерованными кругами 36px, соединительными линиями, иконками шагов и подписями
- Строка поиска пациента 48px с autocomplete dropdown (rich results: аватар, ФИО, телефон, дата рождения)
- Карточки врачей с фото, ФИО, специализацией и badge ближайшего слота
- Calendar picker с зелёными точками доступности
- Time slot pills — кнопки 40px с временем, три состояния: свободный (синий outline), занятый (серый disabled), выбранный (синий filled)
- Summary-карточка с парами label-value, разделёнными линиями #F1F5F9
- SMS/WhatsApp checkbox с кастомным стилем (синяя галочка)

UX-аспекты:
- Автоматический переход на следующий шаг после выбора на текущем (с небольшой задержкой 300ms для визуальной обратной связи)
- Навигация назад сохраняет все ранее выбранные данные (state persistence)
- Calendar автоматически выбирает ближайший доступный день при загрузке шага 3
- При выборе слота — мгновенный переход к шагу 4 (подтверждение)
- Autocomplete поиска работает по ФИО, телефону, номеру карты — поиск начинается после 2 символов
- Если пациент зарегистрирован только что (шаг 1 → регистрация), возвращается на шаг 2 с предзаполненным пациентом
- Кнопка "Записать" показывает spinner при отправке, затем success-экран с деталями и кнопкой "Новая запись"

Брендинг:
- Progress indicator в фирменном синем #1E40AF
- Summary-карточка подтверждения — чистая, как бланк записи с логотипом клиники
- Минималистичный стиль без лишних декораций — фокус на быстрой записи

Доступность:
- Progress bar с aria-label для каждого шага ("Шаг 1 из 4: Выбор пациента — завершён/текущий/впереди")
- Calendar полностью navigable с клавиатуры (стрелки для дней, PageUp/PageDown для месяцев)
- Time slots с aria-labels ("14:30 — свободно" / "15:00 — занято")
- Focus management: при переходе на новый шаг фокус перемещается на первый интерактивный элемент
- Autocomplete dropdown с role="listbox", результаты с role="option"
- Contrast ratio всех текстов не менее 4.5:1 на соответствующих фонах

Тренды:
- Анимированные переходы между шагами (slide left при "Далее", slide right при "Назад")
- Пульсация зелёных точек на днях с доступными слотами в calendar
- Glassmorphism на summary-карточке подтверждения (backdrop-blur, полупрозрачный фон)
- Progress indicator с плавным fill-анимацией линий при переходе между шагами
- Confetti-like микроанимация при успешной записи
```

## 5.3 Регистрация нового пациента

```
Разработайте современный, профессионально выглядящий интерфейс комплексной формы регистрации нового пациента для рецепшена в медицинской информационной системе.

Визуальный стиль:
- Чистый form-дизайн с чётким разделением на секции, визуальная иерархия через карточки-секции
- Основная палитра: primary #1E40AF, фон #F8FAFC, обязательные поля — asterisk #EF4444, success #10B981, ошибки #EF4444
- Каждая секция — белая карточка (shadow-sm, border-radius 12px, padding 24px), между секциями gap 16px
- Типографика Inter: заголовки секций 18px bold с иконкой слева, labels 14px semibold text-gray-700, placeholder 14px regular text-gray-400
- Иконки Lucide/Phosphor для заголовков секций и некоторых полей

Макет и структура:
- Заголовок страницы: "Регистрация пациента" 24px bold, breadcrumb "Дашборд → Пациенты → Регистрация"
- Секция 1 — "Личные данные" (иконка User): grid 3 колонки. Поля: Фамилия* (text input), Имя* (text input), Отчество (text input) — первый ряд. Дата рождения* (date picker с popup-календарём, формат ДД.ММ.ГГГГ, автоматический расчёт возраста справа от поля), Пол* (radio group: Мужской/Женский, кастомные radio-кнопки с иконками Mars/Venus) — второй ряд. Фото пациента: зона drag-drop (border dashed #D1D5DB, border-radius 12px, 120×120px) с иконкой Camera по центру, текст "Загрузить фото" + кнопка "Сделать фото" (открывает webcam-модал с preview и кнопкой захвата)
- Секция 2 — "Контакты" (иконка Phone): Телефон* — input с маской +998 (XX) XXX-XX-XX, иконка флага Узбекистана слева, автоформат при вводе. Email — input type email с иконкой Mail. Адрес — составной блок: город (select с популярными + текст), улица (text), дом (text), квартира (text) — в одну строку grid 4 колонки
- Секция 3 — "Документы" (иконка FileText): Тип документа (select: Паспорт, ID-карта, Свидетельство о рождении). Серия и номер (text input с маской в зависимости от типа). Кем выдан (text). Дата выдачи (date picker). Зона загрузки скана: drag-drop область (border dashed, иконка Upload, текст "Перетащите скан или нажмите для выбора"), preview загруженного файла (thumbnail + имя файла + кнопка удаления)
- Секция 4 — "Медицинская информация" (иконка Heart): Группа крови (select dropdown: I+, I-, II+, II-, III+, III-, IV+, IV-). Аллергии (tag input: при вводе текста + Enter создаётся chip/tag с текстом и кнопкой × для удаления; цвет тегов #FEF2F2 фон + #EF4444 текст). Заметки (textarea, 3 строки, placeholder "Хронические заболевания, особые указания...")
- Секция 5 — "Страхование / ДМС" (иконка Shield): Страховая компания (select с autocomplete из справочника). Номер полиса (text input). Действует до (date picker). Если полис не заполнен — секция визуально свёрнута (collapsed), кнопка "Добавить полис ДМС" раскрывает
- Секция 6 — "Согласия": checkbox "Согласие на обработку персональных данных"* — кастомный checkbox (border-radius 4px, checked = #1E40AF фон + белая галочка), рядом дата автозаполнения (текущая дата, 12px gray). Checkbox "Информированное добровольное согласие". Оба checkbox обязательны, при незаполнении — красная рамка
- Footer (sticky bottom или в конце формы): кнопка "Сохранить" (primary #1E40AF, 44px высота, padding 0 32px) + кнопка "Сохранить и распечатать" (secondary outline, иконка Printer) — печатает пакет документов (договор + согласие ПДн + ИДС)

Элементы UI:
- Form fields — высота 40px, border 1px #D1D5DB, border-radius 8px, focus — border #1E40AF + ring #1E40AF/20%, error — border #EF4444 + текст ошибки 12px #EF4444 под полем
- Labels над полями, 14px semibold, обязательные с красной звёздочкой *
- Phone input с маской и иконкой флага
- Date picker с popup-календарём, формат ДД.ММ.ГГГГ
- Radio кнопки кастомные (круг 20px, selected = #1E40AF filled)
- Tag input для аллергий: поле ввода + chips/tags ниже, каждый tag — pill с текстом и × для удаления
- Drag-drop зоны для фото и скана с dashed border и иконкой
- Collapsible секция страхования с toggle-кнопкой
- Checkbox кастомный — квадрат 20px, checked = #1E40AF с белой галочкой

UX-аспекты:
- Inline-валидация: ошибка появляется при blur (потере фокуса), исчезает при исправлении
- Обязательные поля валидируются перед сохранением; при ошибке — scroll к первому невалидному полю + focus
- Autofocus на первом поле (Фамилия) при открытии формы
- Tab-навигация по всем полям в логическом порядке
- Телефон автоформатируется при вводе цифр (маска +998)
- "Сохранить и распечатать" — сохраняет + открывает print-preview пакета документов
- При дубликате (по ФИО+дата рождения или телефону) — warning-alert "Возможно, пациент уже зарегистрирован" со ссылкой на существующую карточку

Брендинг:
- Заголовки секций с иконками в фирменном синем #1E40AF
- Кнопки в фирменном синем, secondary — outline
- Чистый медицинский стиль без лишнего декора

Доступность:
- Все поля с явными <label> элементами, связанными через htmlFor/id
- Обязательные поля с aria-required="true"
- Сообщения об ошибках связаны через aria-describedby
- Tag-chips в аллергиях удаляемые клавишей Backspace и кнопкой ×, каждый tag focusable
- Radio-группа пол с role="radiogroup" и aria-label
- Drag-drop зоны с альтернативной кнопкой "Выбрать файл" для клавиатурной навигации

Тренды:
- Плавная анимация валидации (shake на ошибке, fade-in для сообщений об ошибках)
- Neumorphic checkboxes — мягкая тень + вдавленный вид при checked
- Webcam-модал для фото с современным UI (затемнённый фон, камера по центру, кнопка-круг для захвата)
- Smooth collapse/expand анимация для секции страхования
- Subtle border-color transition на focus полей (200ms ease)
```

## 5.4 Расписание: обзор всех врачей

```
Разработайте современный, профессионально выглядящий интерфейс обзора расписания всех врачей в формате Gantt-диаграммы для рецепшена в медицинской информационной системе.

Визуальный стиль:
- Gantt-подобная дневная сетка: строки — врачи, столбцы — временные слоты по 15 минут
```
