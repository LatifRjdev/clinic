# Общие компоненты — Design Prompts (Stitch)

> Переиспользуемые UI-компоненты, применяемые во всех ролях и модулях МИС клиники.
> Дизайн-система: Primary #1E40AF, Background #F8FAFC, Success #10B981, Danger #EF4444, Warning #F59E0B.
> Шрифт Inter. Иконки Lucide/Phosphor. Платформа: web, desktop-first, адаптив для пациентского портала.

## 9.1 Sidebar навигация

```
Разработайте современный, профессионально выглядящий компонент боковой навигации (sidebar) для медицинской информационной системы крупной многопрофильной клиники.

Визуальный стиль:
- Тёмный вариант sidebar: фон #1E293B, текст пунктов меню #CBD5E1, активный пункт — белый текст #FFFFFF на подложке rgba(30,64,175,0.25). Светлый вариант для альтернативной темы: фон #FFFFFF, текст #475569, активный — #1E40AF на подложке #EFF6FF. Два состояния ширины: развёрнутый 240px и свёрнутый 64px (только иконки). Шрифт Inter 14px regular для пунктов, 12px caption для подписей групп, иконки Lucide 20px. Разделительные линии между группами пунктов — rgba(255,255,255,0.08) для тёмного варианта. Общее ощущение — строгий, медицинский, но современный интерфейс.

Макет и структура:
- Верхний блок: логотип клиники — полная версия (иконка + название) в развёрнутом состоянии, только иконка-символ в свёрнутом. Плавная анимация перехода между вариантами логотипа (crossfade 200ms). Отступ от верхнего края 16px, от боков 16px. Под логотипом — тонкий разделитель.
- Основная навигация: вертикальный список пунктов меню. Каждый пункт — иконка (20px) + текст + опциональный badge-счётчик справа (красный кружок с числом для уведомлений, синий для количества элементов). Активный пункт выделяется: левая вертикальная полоса 3px цвета #1E40AF + фоновый тинт + текст и иконка становятся ярче. Hover-состояние: фон rgba(255,255,255,0.05) для тёмного варианта, плавный переход 150ms.
- Вложенные подменю: раскрываются по клику на родительский пункт, chevron-иконка поворачивается на 90° при раскрытии (анимация 200ms). Вложенные пункты с дополнительным отступом 12px слева, шрифт 13px. Состояние раскрытия подменю сохраняется в localStorage.
- Группировка пунктов: метки-секции (например «Основное», «Управление», «Настройки») — 11px uppercase tracking-wider, цвет muted, отступ сверху 20px.
- Нижний блок (footer): мини-карточка пользователя — аватар 32px (круглый, с fallback-инициалами при отсутствии фото) + ФИО (14px, одна строка, ellipsis) + роль (12px muted). В свёрнутом состоянии — только аватар. Ниже — кнопка сворачивания sidebar (иконка ChevronLeft/ChevronRight), отделённая разделителем.
- Меню отличается по ролям: разные пункты видны владельцу, администратору, врачу, регистратору, пациенту. Ролевая фильтрация на уровне конфигурации маршрутов.

Элементы UI:
- Плавная анимация сворачивания/разворачивания: ширина анимируется 200ms ease-in-out, текст пунктов меню fade-out при сворачивании (opacity transition 150ms). В свёрнутом режиме при наведении на иконку — всплывающая подсказка (tooltip) справа с названием пункта и подпунктов. Badge-счётчики: пульсирующая анимация для новых уведомлений. Аватар пользователя с цветным статус-индикатором (зелёная точка — онлайн).

UX-аспекты:
- Активный пункт автоматически определяется по текущему маршруту (URL). При навигации — мгновенная смена active-состояния. Подменю запоминает открытое/закрытое состояние между сессиями. В свёрнутом режиме tooltip появляется с задержкой 200ms для предотвращения мерцания. На планшетах (ширина < 1024px): sidebar становится overlay-панелью, появляется по нажатию hamburger-кнопки, закрывается по клику на backdrop.

Доступность:
- Семантическая разметка: <nav aria-label="Основная навигация">, <ul role="list">, <li>. Подменю: aria-expanded="true/false" на кнопке раскрытия. Активный пункт: aria-current="page". Полная клавиатурная навигация: Tab переключает пункты, Enter/Space активирует, стрелки вверх/вниз перемещают фокус, стрелка вправо раскрывает подменю, стрелка влево сворачивает. В свёрнутом режиме tooltip доступен через фокус (не только hover). Контрастность текста на тёмном фоне — минимум WCAG AA (4.5:1).

Тренды:
- Тонкий glassmorphism-эффект на тёмном sidebar: полупрозрачность фона с backdrop-filter: blur(20px) при скролле контента за sidebar. Плавные CSS-переходы ширины с cubic-bezier(0.4,0,0.2,1). Микроанимация иконок при наведении — лёгкий scale(1.05). Скруглённые углы active-подложки (8px). Градиентная подсветка active-пункта вместо сплошной заливки.
```

## 9.2 Хедер (Header Bar)

```
Разработайте современный, профессионально выглядящий компонент верхней панели (header bar) для медицинской информационной системы крупной многопрофильной клиники.

Визуальный стиль:
- Фиксированная верхняя панель высотой 56px, sticky-позиционирование. Фон: белый #FFFFFF с glassmorphism-эффектом — rgba(255,255,255,0.85) с backdrop-filter: blur(12px). Тень при скролле: 0 1px 3px rgba(0,0,0,0.05), усиливающаяся до 0 4px 12px rgba(0,0,0,0.08) при прокрутке страницы. Текст #334155, иконки #64748B, hover-иконки #1E40AF. Шрифт Inter. Разделительная линия нижняя: 1px solid #E2E8F0. Общая эстетика — лёгкость, воздушность, ненавязчивость.

Макет и структура:
- Левая часть: хлебные крошки (breadcrumbs) или заголовок текущей страницы (16px semibold). Breadcrumbs: «Главная > Раздел > Страница» — промежуточные элементы кликабельны (#64748B, hover: #1E40AF с underline), текущая страница — bold #1E293B, некликабельная. Разделитель «/» цвета #CBD5E1. На мобильных — только заголовок страницы с кнопкой «Назад».
- Центральная часть: глобальный поиск — input-поле шириной 400px (max), скруглённые углы 10px, фон #F1F5F9, иконка лупы слева, справа — подсказка горячей клавиши «⌘K» в виде мини-badge (фон #E2E8F0, текст #64748B, 11px, border-radius 4px). Placeholder: «Поиск пациентов, врачей, услуг...». По клику или Cmd+K — открывается модальное окно поиска (search modal): поле ввода сверху (автофокус), ниже — блок «Недавние запросы» (иконка часов + текст, кликабельно), затем — категоризированные результаты (секции «Пациенты», «Врачи», «Услуги», «Записи» с иконками и заголовками секций), навигация стрелками вверх/вниз, Enter для перехода.
- Правая часть (flex-gap 12px): (1) Колокольчик уведомлений — иконка Bell 20px, красный badge-счётчик в правом верхнем углу (минимум 16px, шрифт 10px bold, пульсация для новых). По клику — выпадающая панель уведомлений (ширина 380px, max-height 480px, скролл): заголовок «Уведомления» + ссылка «Отметить все прочитанными», список карточек уведомлений (иконка типа + текст + время relative + точка непрочитанного), кнопка «Все уведомления» внизу. (2) Переключатель языка RU/UZ — сегментированные pill-кнопки (28px высота, border-radius 6px, активный — фон #1E40AF текст белый, неактивный — фон transparent текст #64748B). (3) Аватар пользователя 32px круглый — по клику dropdown-меню: блок ФИО + роль + email сверху, пункты «Мой профиль», «Настройки», разделитель, «Сменить филиал» (если multi-branch), разделитель, «Выйти» (красный текст).

Элементы UI:
- Glassmorphism-эффект фона при скролле — плавный переход прозрачности (transition 300ms). Search modal с overlay (backdrop-filter: blur(4px)), анимация появления — scale(0.98) + opacity 0 → scale(1) + opacity 1 за 200ms. Dropdown-панели: появление fade-in + translateY(-4px→0) за 150ms, тень shadow-xl. Badge уведомлений: красный #EF4444, белый текст, pulse-анимация (keyframes scale 1→1.2→1). Переключатель языка: animated slider-подложка перемещается между RU/UZ (transition 200ms).

UX-аспекты:
- Поиск: debounce 300ms при вводе, мгновенные результаты, подсветка совпадений bold, Escape закрывает modal. Уведомления: непрочитанные выделены фоном #F0F9FF, клик по уведомлению — переход к контексту + автоматическая пометка прочитанным. Dropdown закрывается по клику вне области. Breadcrumbs генерируются автоматически из маршрута. Подсказка ⌘K обучает пользователя горячей клавише.

Доступность:
- Search input: скрытый визуально label с aria-label="Глобальный поиск", результаты в role="listbox", навигация aria-activedescendant. Уведомления: кнопка aria-label="Уведомления, N непрочитанных", панель role="dialog". Dropdown-меню: role="menu", пункты role="menuitem", навигация стрелками. Breadcrumbs: <nav aria-label="Хлебные крошки">, <ol>. Все интерактивные элементы доступны с клавиатуры, фокус-кольцо 2px #1E40AF.

Тренды:
- Glassmorphism header с backdrop-filter: blur(12px) — прозрачность усиливается при скролле. Command palette стиль поиска (как в VS Code / Linear). Анимированная тень header при скролле. Pill-переключатель языка с sliding indicator. Микроанимация иконки колокольчика при появлении нового уведомления (лёгкое покачивание).
```

## 9.3 Таблица данных (DataTable)

```
Разработайте современный, профессионально выглядящий компонент таблицы данных (DataTable) для медицинской информационной системы крупной многопрофильной клиники.

Визуальный стиль:
- Чистая, высокочитаемая таблица с выверенной типографикой. Фон заголовка #F8FAFC, фон строк чередующийся: белый #FFFFFF и #FAFBFC (striped, опционально). Границы: горизонтальные разделители #E2E8F0 между строками, без вертикальных линий. Hover-строка: фон #F1F5F9 с плавным переходом 150ms. Выделенная строка (selected): фон #EFF6FF с левой границей 3px #1E40AF. Padding ячеек: 12px 16px. Шрифт Inter 14px regular для данных, заголовки 13px semibold uppercase, letter-spacing 0.05em, цвет #64748B. Иконки сортировки #94A3B8, активная — #1E40AF.

Макет и структура:
- Панель инструментов (toolbar) над таблицей: поле поиска (input, иконка лупы, placeholder «Поиск...», 280px), кнопки фильтров (иконка Filter + «Фильтры» с badge-счётчиком активных), dropdown «Столбцы» для управления видимостью колонок (чекбоксы для каждой), кнопка экспорта — dropdown с пунктами: CSV, Excel (.xlsx), PDF, печать (иконки для каждого формата). Справа — основная кнопка действия «+ Создать» (primary).
- Заголовок таблицы (thead): sticky при вертикальном скролле, z-index выше контента. Каждая колонка: текст + иконка сортировки (нейтральная ↕, по возрастанию ↑, по убыванию ↓, клик циклически переключает). Колонка чекбоксов (крайняя левая, 48px) — чекбокс «выбрать все» с тремя состояниями (unchecked, checked, indeterminate). Колонки поддерживают разные типы данных: текст, число (выравнивание вправо), дата, badge-статус (цветной), аватар+имя, кнопки действий.
- Тело таблицы: строки с hover-подсветкой. Раскрываемые строки (expandable): клик на chevron-иконку раскрывает детальную область под строкой с плавной анимацией высоты (200ms). Первая колонка (или первые 1-2) — sticky при горизонтальном скролле с тенью-разделителем справа.
- Per-column фильтры: выпадающие dropdown-фильтры в каждом заголовке столбца (иконка воронки при наведении) — текстовый поиск, диапазон дат, выбор из списка в зависимости от типа данных.
- Перетаскивание колонок (drag reorder): grab-курсор на заголовке, ghost-элемент при перетаскивании, drop-зона с индикатором.
- Изменение ширины колонок (column resize): тонкая линия-разделитель между заголовками (4px drag-зона), курсор col-resize, визуальная линия-индикатор при перетаскивании.
- Подвал таблицы (footer): слева — «Показано 1–20 из 342 записей», справа — пагинация: кнопки страниц (< 1 2 3 ... 17 >), выбор количества строк на странице (select: 20 / 50 / 100). Активная страница — фон #1E40AF, текст белый, остальные — фон transparent.
- Панель массовых действий (bulk actions): появляется при выборе строк — горизонтальная полоса над таблицей с фоном #EFF6FF: «Выбрано N записей» + кнопки действий (Удалить — красная, Экспорт, другие контекстные) + ссылка «Снять выделение».
- Пустое состояние: центрированная иллюстрация (line-art) + текст «Данные не найдены» + кнопка CTA. Состояние загрузки: skeleton-строки (8 штук) с shimmer-анимацией, повторяющие структуру колонок.
- Адаптивность: на ширине < 768px — горизонтальный скролл таблицы, первая колонка frozen (sticky left: 0, фон белый, тень справа). Альтернативный вариант — переключение в card-view (каждая строка = карточка с label:value парами).

Элементы UI:
- Иконки сортировки с анимацией перехода между состояниями (rotate/fade 150ms). Чекбоксы: кастомные, 18px, border-radius 4px, checked — фон #1E40AF с галочкой белой. Раскрываемая строка: chevron поворачивается на 180° при раскрытии. Skeleton-загрузка: прямоугольники с shimmer linear-gradient(90deg, #F1F5F9, #E2E8F0, #F1F5F9), анимация 1.5s infinite. Пагинация: кнопки 32px, border-radius 8px, hover-фон #F1F5F9.

UX-аспекты:
- Клик на заголовок — сортировка (none → asc → desc → none). Shift+клик — мультиколоночная сортировка. Drag-resize колонок сохраняется в localStorage. Выбор «все на странице» vs «все записи по фильтру» — уточняющее сообщение. Поиск — debounce 300ms. Клавиатура: стрелки для навигации по строкам, Space — выбрать, Enter — раскрыть/перейти.

Доступность:
- Семантика: <table>, <thead>, <tbody>, <th scope="col">, <td>. Сортировка: aria-sort="ascending|descending|none". Чекбоксы: aria-label="Выбрать строку [имя]". Пагинация: <nav aria-label="Пагинация">, текущая страница aria-current="page". Раскрываемые строки: aria-expanded. Screen reader: объявление общего количества строк и выбранных. Фокус-кольцо 2px #1E40AF на всех интерактивных элементах.

Тренды:
- Мягкие hover-переходы (background 150ms ease). Skeleton shimmer с плавным градиентом. Rounded-углы кнопок пагинации. Sticky-заголовок с лёгкой тенью при скролле. Панель bulk actions с slide-down анимацией. Card-view на мобильных с мягкими тенями. Drag-and-drop колонок с ghost-элементом (opacity 0.6, тень).
```

## 9.4 Формы и поля ввода

```
Разработайте современный, профессионально выглядящий набор компонентов форм и полей ввода для медицинской информационной системы крупной многопрофильной клиники.

Визуальный стиль:
- Единообразный стиль всех полей: высота 42px (однострочные), border-radius 10px, border 1.5px solid #E2E8F0, фон #FFFFFF, padding 10px 14px. Шрифт Inter 14px regular для значений, label 14px medium #334155 над полем, placeholder #94A3B8, helper text 12px #64748B под полем, error text 12px #EF4444 под полем с иконкой ошибки. Расстояние между полями (gap) — 8px между label и input, 16px между полями. Цветовая система состояний: default border #E2E8F0, hover border #CBD5E1, focus border #1E40AF + ring 3px rgba(30,64,175,0.15), filled border #CBD5E1, error border #EF4444 + ring rgba(239,68,68,0.15) + красный текст ошибки, disabled фон #F8FAFC + border #E2E8F0 + текст #94A3B8, readonly фон #F8FAFC без курсора.

Макет и структура:
- Text input: label сверху (обязательные — красная звёздочка *), placeholder внутри, helper text под полем. Поддержка prefix (иконка или текст слева внутри поля, например иконка лупы) и suffix (иконка или текст справа, например единица измерения «кг»). Очистка по иконке X при заполненном поле.
- Number input: те же стили + кнопки +/- по бокам (правая сторона, вертикальный стек или обе стороны). Min/max ограничения, шаг (step). Выравнивание текста вправо.
- Phone input: маска +998 (XX) XXX-XX-XX, фиксированный prefix «+998» с иконкой флага Узбекистана слева (серый фон зоны prefix). Автоматическое форматирование при вводе. Валидация длины.
- Email input: иконка конверта слева, валидация формата на blur, зелёная галочка при валидном адресе.
- Password input: кнопка toggle show/hide (иконка Eye/EyeOff) справа. Индикатор сложности пароля под полем — полоска из 4 сегментов (красный/оранжевый/жёлтый/зелёный) + текст («Слабый», «Средний», «Хороший», «Сильный»).
- Textarea: auto-grow (min-height 80px, max-height 200px), счётчик символов в правом нижнем углу «145/500», border-radius 10px.
- Date picker: input с иконкой календаря справа, по клику — popup-календарь (месяц/год навигация, дни сетка 7 колонок, сегодня выделен кольцом, выбранный — фон #1E40AF белый текст, диапазон — подсветка между датами). Кнопки «Сегодня» и «Очистить» в footer popup.
- Time picker: input с иконкой часов, dropdown со списком времени шагом 15/30 минут или ручной ввод HH:MM.
- Date range picker: два связанных поля «От» — «До» с единым popup-календарём, визуальный диапазон между выбранными датами (фон-полоса #EFF6FF).
- Select dropdown (single): кастомный dropdown, иконка ChevronDown справа, поиск внутри при > 7 опциях, очистка выбора X. Выбранное значение отображается в поле. Dropdown: max-height 280px, скролл, hover-подсветка пунктов.
```

## 9.12 Статус-бейджи и тэги

```
Разработайте современный, профессионально выглядящий набор статус-бейджей и тэгов для медицинской информационной системы крупной многопрофильной клиники.

Визуальный стиль:
- Единый стиль: border-radius 6px, шрифт Inter 12px medium, padding 4px 10px, line-height 1. Цветовая схема — «светлый фон + насыщенный текст» (tinted background): фон — основной цвет с opacity 0.1, текст — основной цвет с полной насыщенностью (darken). Это обеспечивает читаемость и мягкость. Без border (или тонкий border 1px того же цвета с opacity 0.2 для усиления). Высота: 24px (standard), 20px (compact). Иконка опционально слева (14px, в цвете текста), dot-индикатор (6px кружок) опционально слева.

Макет и структура:
- Статусы записей (appointment statuses): «Запланировано» — синий (#1E40AF фон rgba(30,64,175,0.1), текст #1E40AF), «Подтверждено» — зелёный (#10B981 фон rgba(16,185,129,0.1), текст #047857), «Ожидание» — жёлтый (#F59E0B фон rgba(245,158,11,0.1), текст #B45309), «В процессе» — фиолетовый (#8B5CF6 фон rgba(139,92,246,0.1), текст #6D28D9), «Завершено» — серый (#64748B фон rgba(100,116,139,0.1), текст #475569), «Отменено» — красный (#EF4444 фон rgba(239,68,68,0.1), текст #DC2626), «Неявка» — тёмный (#1E293B фон rgba(30,41,59,0.1), текст #1E293B). Каждый статус имеет dot-индикатор (6px кружок в цвете текста) слева от текста.
- Приоритеты: «Низкий» — серый (#94A3B8 тинт), «Обычный» — синий (#1E40AF тинт), «Высокий» — оранжевый (#F97316 фон rgba(249,115,22,0.1), текст #EA580C), «Срочный» — красный (#EF4444 тинт) с CSS-анимацией пульсации (pulse: box-shadow 0 0 0 2px rgba(239,68,68,0.3) → 0 0 0 6px rgba(239,68,68,0) infinite 1.5s). Иконка AlertCircle для «Срочный».
- Роли пользователей: «Владелец» — фон indigo тинт, «Администратор» — синий тинт, «Врач» — зелёный тинт со стетоскопом-иконкой, «Регистратор» — голубой тинт, «Бухгалтер» — amber тинт, «Пациент» — серый тинт. Каждый ролевой badge — с характерной иконкой слева (14px).
- Способы оплаты: мини-бейджи с иконками — «Наличные» (иконка Banknote + текст), «Карта» (иконка CreditCard), «Перевод» (иконка ArrowRightLeft), «Страховка» (иконка Shield). Нейтральный серо-синий стиль (фон #F1F5F9, текст #475569).
- Общие тэги (tags): цветные pill-элементы для категоризации (диагнозы, услуги, отделения). Набор из 8 цветов (blue, green, red, yellow, purple, pink, cyan, orange) — каждый тег получает цвет. Removable-вариант: кнопка X (12px) справа внутри тега, hover на X — фон тега становится насыщеннее. Non-removable — без X, обычный display-only. Поддержка нескольких тегов в строку (flex-wrap, gap 4px). Размеры: default (24px), small (20px, 11px font).
- Dot-status (точечный статус): минимальный индикатор — только цветной кружок 8px (зелёный — онлайн/активен, серый — офлайн, жёлтый — занят, красный — недоступен) рядом с аватаром или именем. Для critical статусов — пульсация.

Элементы UI:
- Dot-индикаторы: 6px для inline, 8px для standalone, 10px для аватаров. Pulse-анимация для «Срочный» и «Онлайн» (keyframes scale 1→1.5 + opacity 1→0, 1.5s infinite). Removable tags: X-кнопка с hover-переходом (фон тега darkens 10%). Transition при смене статуса: crossfade цвета (background-color transition 300ms, color transition 300ms). Tag появление: scale(0.9→1) + opacity(0→1) за 150ms. Tag удаление: scale(1→0.9) + opacity(1→0) за 150ms.

UX-аспекты:
- Статус-бейджи — только для отображения (не кликабельны). Тэги removable — кнопка X удаляет тег (с подтверждением или undo toast). Единая цветовая система статусов по всей системе — пациент видит те же цвета что и врач/администратор. Размер бейджа — compact в таблицах (20px), standard в карточках и детальных view (24px). Tooltip при наведении на badge — полное описание статуса (если текст сокращён). Максимум 5 тэгов в строку, далее «+N ещё» (клик раскрывает все).

Доступность:
- Каждый badge: текст содержит полное значение (не только цвет). Screen reader: aria-label включает тип + значение (например aria-label="Статус: Подтверждено"). Dot-status: span с aria-label="Статус: Онлайн" (скрытый визуально текст или sr-only span). Removable tags: кнопка X — aria-label="Удалить тег [название]". Pulse-анимация: prefers-reduced-motion: reduce отключает анимацию. Цветовая индикация не является единственным способом различия — текст и иконки дублируют смысл. Контрастность текста на tinted-фоне — минимум WCAG AA (4.5:1 для 12px текста).

Тренды:
- Tinted background стиль (популярен в Linear, Notion, GitHub): мягкие подложки вместо solid-цветов. Dot-индикаторы с pulse для live-статусов. Rounded pill-форма (6px, не fully rounded — не capsule). Compact-размеры для data-dense интерфейсов. Smooth color transitions при смене статуса. Микро-иконки внутри бейджей для дополнительного семантического значения. Группировка тэгов с overflow-counter (+N).
```

---

> Все компоненты следуют единой дизайн-системе: Primary #1E40AF, шрифт Inter, иконки Lucide/Phosphor, border-radius 10-16px, высота контролов 42px. Состояния (hover, focus, active, disabled, error) анимируются через CSS transitions 150-200ms. Доступность: WCAG AA, семантическая разметка, клавиатурная навигация, aria-атрибуты.
