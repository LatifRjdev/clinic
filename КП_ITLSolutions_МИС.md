## О решении

Предлагаем вашему вниманию комплексное решение для автоматизации управления многопрофильной клиникой. **Единая веб-платформа** объединяет все ключевые процессы клиники — от онлайн-записи пациента до ведения электронной медицинской карты, управления финансами и аналитики.

Система работает в облаке (или на сервере клиники) и доступна с любого устройства через браузер — компьютер, планшет или телефон. Не нужно устанавливать программы — достаточно открыть сайт и войти в систему.

Для каждого сотрудника — врача, медсестры, рецепшен, бухгалтера — предусмотрен свой интерфейс с нужными функциями. Пациенты могут записываться онлайн и просматривать результаты приёмов.

## Роли системы

Каждый пользователь видит только те функции, которые ему нужны:

| Роль | Что видит и делает |
|------|-------------------|
| **Владелец клиники** | Финансы, персонал, аналитика, прайс-лист, маркетинг — полный контроль бизнеса |
| **Главный врач** | Расписание врачей, контроль качества лечения, статистика приёмов |
| **Врач** | Ведение приёма: осмотр, диагноз (МКБ-10), назначения, рецепты, направления |
| **Медсестра** | Процедуры, витальные показатели (давление, пульс, температура), расходные материалы |
| **Рецепшен** | Запись пациентов, управление очередью, оплата, печать документов |
| **Бухгалтер** | Доходы/расходы, зарплаты, ДМС-расчёты, налоги, отчёты |
| **Сисадмин** | Пользователи, роли, интеграции, бэкапы, логи, настройки |
| **Пациент** | Онлайн-запись, просмотр результатов, общение с врачом |

## Функциональные модули системы

### Пациенты

| Функция | Описание простыми словами |
|---------|--------------------------|
| Регистрация | Карточка пациента с ФИО, контактами, паспортом, полисом, аллергиями |
| Поиск | Быстрый поиск по имени, телефону, номеру карты или паспорту |
| Полная история | Все визиты, диагнозы, назначения, счета — в одном месте |
| Импорт/экспорт | Загрузка списка пациентов из Excel/CSV и выгрузка обратно |
| Объединение дублей | Если пациент записан дважды — данные объединяются без потери |

### Расписание и запись

| Функция | Описание простыми словами |
|---------|--------------------------|
| График врачей | Настройка рабочих дней, часов, перерывов, длительности приёма |
| Запись на приём | Рецепшен или сам пациент выбирает врача, дату и время |
| Визуальное расписание | Удобный календарь: вид по дню, неделе, месяцу |
| Живая очередь | Экран в клинике показывает, кто следующий |
| Напоминания | SMS за 24 часа и 2 часа до приёма |
| Управление кабинетами | Проверка, чтобы два врача не оказались в одном кабинете |

### Электронная медицинская карта (ЭМК)

| Функция | Описание простыми словами |
|---------|--------------------------|
| Ведение приёма | Врач заполняет: жалобы → анамнез → осмотр → диагноз → рекомендации |
| Шаблоны осмотров | Готовые формы по специальностям — терапевт, хирург, ЛОР и т.д. |
| Диагнозы МКБ-10 | Международная классификация болезней с поиском по коду или названию |
| Рецепты | Назначение медикаментов с дозировкой и курсом, печать рецепта в PDF |
| Направления | Направление к другому специалисту или на обследование |
| Электронная подпись | Врач подписывает запись — после этого она защищена от изменений |
| Витальные показатели | Давление, пульс, температура, вес, рост, сахар — с графиками изменений |
| Прикрепление файлов | Снимки, результаты анализов — всё хранится в карте пациента |

### Биллинг и касса

| Функция | Описание простыми словами |
|---------|--------------------------|
| Каталог услуг | Список всех услуг клиники с ценами и категориями |
| Автоматический счёт | После приёма система сама формирует счёт из оказанных услуг |
| Способы оплаты | Наличные, карта, перевод, ДМС |
| Скидки | Процент или фиксированная сумма — для отдельных пациентов или акций |
| Касса | Открытие/закрытие смены, инкассация, дневной отчёт |
| Печать чеков | Формирование чека в PDF для печати |
| Возвраты | Оформление возврата с подтверждением |

### Финансы

| Функция | Описание простыми словами |
|---------|--------------------------|
| Доходы и расходы | Учёт всех денежных потоков по категориям |
| Зарплаты | Расчёт: оклад + процент от услуг. Ведомости, утверждение, выплата |
| ДМС-расчёты | Реестры услуг для страховых компаний, проверка покрытия полиса |
| Налоговый отчёт | Автоматический расчёт налогов |
| Контрагенты | Поставщики, арендодатели — учёт взаиморасчётов |
| Финансовые отчёты | Прибыли и убытки (P&L), движение денег (Cash Flow), по услугам, по врачам |

### Склад

| Функция | Описание простыми словами |

## Архитектура системы

Система построена на современной клиент-серверной архитектуре. Веб-приложение адаптируется под размер экрана и корректно работает на компьютере, планшете и телефоне.

| Компонент | Описание |
|-----------|---------|
| Веб-приложение | React — современный интерфейс для всех устройств |
| Серверная часть | NestJS (TypeScript) — бизнес-логика и REST API |
| База данных | PostgreSQL — надёжная база данных |
| Кэширование | Redis — ускорение работы и защита от перегрузок |
| Хранилище файлов | MinIO — хранение снимков, документов, результатов анализов |
| Реальное время | WebSocket (Socket.io) — чат, уведомления, очередь |
| Инфраструктура | Docker + Nginx — развёртывание на сервере клиники или в облаке |

## Этапы внедрения решения

1. **Первичная консультация и сбор требований.** Обсуждение задач и целей внедрения, анализ текущих процессов клиники, согласование сценариев использования.

2. **Заключение договора и внесение предоплаты.** После согласования технических требований подписывается договор, вносится предоплата (50%), что запускает отсчёт сроков проекта.

3. **Разработка и настройка системы.** Создание веб-приложения, настройка серверной части, настройка под специфику клиники, реализация всех согласованных модулей.

4. **Тестирование и согласование.** Комплексное тестирование системы, демонстрация заказчику, корректировка по итогам обратной связи.

5. **Ввод в эксплуатацию.** Перенос данных из старой системы (при необходимости), запуск в рабочем режиме, пользователи получают доступ к функционалу.

6. **Обучение и поддержка.** Проводится инструктаж всех категорий пользователей (рецепшен, врачи, бухгалтерия), предоставляется гарантийная техническая поддержка.

# СРАВНЕНИЕ: Веб-система ITLSolutions vs Текущая локальная программа

## О текущей системе клиники

В настоящее время в клинике используется **локальная программа**, разработанная около 2 лет назад. Она установлена на компьютеры внутри клиники и работает только в локальной сети.

**Единственная функция текущей системы — запись пациентов на приём.** Все остальные процессы (медицинские карты, финансы, склад, аналитика) ведутся вручную — на бумаге, в Excel-таблицах или вовсе не ведутся.

Кроме того, интерфейс программы **морально устарел** — он неудобен, сложен в освоении и визуально напоминает программы начала 2000-х годов. Сотрудники тратят лишнее время на выполнение простых действий из-за неинтуитивного дизайна.

## О системе ITLSolutions

Веб-система ITLSolutions — это **полноценная медицинская информационная система (МИС)**, которая работает через браузер (как Gmail или Instagram). Не нужно ничего устанавливать — открыл сайт, вошёл и работаешь. Система охватывает **все процессы клиники**: от записи пациента до финансовой отчётности.

## Что теряет клиника, используя текущую программу?

Без полноценной системы клиника ежедневно сталкивается с проблемами, которые стоят **времени и денег**:

| Проблема | Последствие | Как решает ITLSolutions |
|----------|-----------|----------------------|
| Нет медицинских карт в системе | Врачи ведут записи на бумаге. Бумаги теряются, почерк нечитаем, нет истории | Полная ЭМК с историей, поиском, печатью |
| Нет кассы и учёта оплат | Непонятно, кто заплатил, сколько, за что. Деньги «теряются» | Прозрачная касса с отчётами |
| Нет финансовых отчётов | Владелец не знает реальную прибыль и расходы | Автоматические отчёты P&L, Cash Flow |
| Нет учёта зарплат | Зарплаты считаются в Excel, легко ошибиться | Автоматический расчёт оклад + % |
| Нет склада | Медикаменты заканчиваются неожиданно, списания не фиксируются | Учёт остатков с предупреждениями |
| Нет аналитики | Управление «вслепую» — нет данных для принятия решений | Дашборды с ключевыми показателями |
| Устаревший интерфейс | Сотрудники тратят лишнее время, ошибаются, не хотят работать в программе | Современный дизайн, простой в освоении |
| Нет онлайн-записи | Пациенты звонят, ждут на линии, иногда уходят к конкурентам | Пациенты записываются сами за 1 минуту |
| Нет SMS-напоминаний | Пациенты забывают о приёме — клиника теряет деньги на пустых слотах | Автоматическое напоминание за 24ч и 2ч |
| Нет контроля действий | Непонятно, кто удалил или изменил запись | Полный журнал аудита |
