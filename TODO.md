## ЧАСТЬ 1: ИСПРАВИТЬ (баги и проблемы)

### 🔴 Критические (система ломается)

- [ ] **1.1** Scheduling: `endTime` = `startTime` при создании записи — запись создаётся с нулевой длительностью
  - Файл: `frontend/src/features/scheduling/SchedulingPage.tsx` ~строка 230
  - Нужно: заменить `endTime: values.startTime` на `endTime: values.endTime`

- [ ] **1.2** Rooms API: пути `findOne`, `update`, `remove` идут на `/rooms/:id` вместо `/scheduling/rooms/:id` → 404
  - Файл: `frontend/src/api/services/scheduling.service.ts` строки 97-106

- [ ] **1.3** EMR контроллеры без `@UseGuards(JwtAuthGuard)` — мед.записи, рецепты, направления доступны без авторизации
  - Файлы: `backend/src/modules/emr/controllers/medical-records.controller.ts`, `prescriptions.controller.ts`, `referrals.controller.ts`

- [ ] **1.4** Billing контроллеры без `@UseGuards(JwtAuthGuard)` — услуги и платежи доступны без авторизации
  - Файлы: `backend/src/modules/billing/controllers/services.controller.ts`, `payments.controller.ts`

- [ ] **1.5** Reception пользователь отсутствует в seed — при пересозданий базы reception пропадёт
  - Файл: `backend/src/seeds/seed.ts`

### 🟡 Серьёзные (работает неправильно)

- [ ] **1.6** Нет валидации `endTime > startTime` — можно создать запись где конец раньше начала
  - Backend: `CreateAppointmentDto` + `AppointmentsService`
  - Frontend: форма создания записи

- [ ] **1.7** Нет проверки расписания врача при записи — можно записать в нерабочий день/час
  - Backend: `AppointmentsService.create()` — проверяет только конфликты, но не расписание

- [ ] **1.8** Invoice и EMR таблицы показывают UUID пациента вместо ФИО
  - Файлы: `frontend/src/features/billing/InvoicesPage.tsx`, `frontend/src/features/emr/EmrPage.tsx`

- [ ] **1.9** Кнопки "Печать" в мед.записях и счетах не реализованы (есть кнопка, нет действия)
  - Файлы: `EmrPage.tsx`, `InvoicesPage.tsx`

- [ ] **1.10** Настройки профиля Reception/Accountant/Nurse — переключатели не сохраняются на backend
  - Файл: `frontend/src/features/profile/ProfilePage.tsx`

- [ ] **1.11** WebSocket чата без обработки ошибок и реконнекта
  - Файл: `frontend/src/features/chat/ChatPage.tsx`

- [ ] **1.12** Нет изоляции по филиалам — все данные видны всем пользователям
  - Backend: все сервисы не фильтруют по `branchId`

- [ ] **1.13** Нет проверки владельца данных — любой врач может менять мед.записи другого врача
  - Backend: EMR module, нет guard-а на data ownership

### 🔵 Мелкие (мешают, но не ломают)

- [ ] **1.14** `useEffect` в `App.tsx` — `fetchProfile` не в dependency array
- [ ] **1.15** Поиск пациентов без debounce — лишние API-запросы при быстром вводе
- [ ] **1.16** Чат: в списке пользователей виден текущий юзер — не исключён из `.filter()`
- [ ] **1.17** AuthStore: при ошибке refresh `user` остаётся старый, хотя `isAuthenticated = false`
